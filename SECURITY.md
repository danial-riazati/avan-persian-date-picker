# Security Policy

## Supported Versions

Avan Persian Date Picker is currently **pre-1.0**. While the project is in
this stage, only the latest published version of each `@avan/*` package
receives security fixes.

| Version            | Supported          |
| ------------------ | ------------------ |
| Latest (0.x)       | :white_check_mark: |
| Older 0.x releases | :x:                |

Once the project reaches a stable 1.0 release, this table will be updated to
reflect a longer-term support policy.

## Reporting a Vulnerability

We take security issues seriously. Please **do not** report security
vulnerabilities through public GitHub issues.

Instead, please report them using one of the following channels, in order of
preference:

1. **GitHub Security Advisories** (preferred): Use the
   ["Report a vulnerability"](https://github.com/danial-riazati/avan-persian-date-picker/security/advisories/new)
   feature under this repository's Security tab. This allows for private
   discussion and coordinated disclosure.
2. **Email**: Send details to **security@avan-datepicker.dev**. Please
   include as much information as possible (see below) to help us triage
   your report quickly.

### What to Include

When reporting a vulnerability, please include, if possible:

- A description of the vulnerability and its potential impact
- Steps to reproduce, or a minimal reproduction (code snippet or repo)
- The affected package(s) and version(s) (e.g. `@avan/core@0.0.0`)
- Any known mitigations or workarounds

## Response Expectations

We aim to acknowledge new vulnerability reports **within 5 business days**.
We will do our best to keep you informed of progress as we investigate and
work on a fix. Response and fix timelines may vary depending on severity and
complexity, since this project does not currently have a dedicated security
team or a formal SLA.

## Disclosure Policy

We follow a **coordinated disclosure** approach:

- Please give us a reasonable amount of time to investigate and address a
  reported vulnerability before any public disclosure.
- Once a fix is available, we will publish a patched release and a
  corresponding GitHub Security Advisory describing the issue, its impact,
  and the affected version range.
- We will credit reporters who wish to be credited, unless anonymity is
  requested.

Thank you for helping keep Avan Persian Date Picker and its users safe.
